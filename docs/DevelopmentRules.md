# 开发规则 (DevelopmentRules)

> 本文件是项目的长期开发纪律，整个项目周期必须遵守，用户无需每次重复说明。

## 1. 项目记忆管理

### 1.1 记忆存放位置
- 项目所有记忆保存在 `docs/` 文件夹，不依赖聊天记录。
- 核心文档：
  - `ProjectMemory.md` — 项目长期记忆与整体状态
  - `CHANGELOG.md` — 变更日志
  - `Roadmap.md` — 路线图
  - `DevelopmentRules.md` — 本文件（开发规则）

### 1.2 开发前必读
每次开始开发前，必须先阅读以下文档，确保掌握项目当前状态：
1. `docs/ProjectMemory.md`
2. `docs/CHANGELOG.md`
3. `docs/Roadmap.md`
4. `docs/DevelopmentRules.md`

## 2. 开发节奏

- **每次只完成一个任务**，完成后立即停止，等用户确认后再继续。
- **不提前实现**用户未明确要求的功能。
- **不擅自修改需求**：严格按用户给定的结构、字段、规则开发。

## 3. 文档更新义务

### 3.1 何时更新
- 每完成一个开发任务、停止开发前，必须自动更新：
  - `docs/ProjectMemory.md`
  - `docs/CHANGELOG.md`
- 如果需求发生变化，必须同步更新 `docs/ProjectMemory.md`。

### 3.2 更新内容要求
更新必须包含：
- 当前版本号
- 本次完成内容
- 已完成功能清单
- 当前开发阶段
- 下一步准备开发内容

### 3.3 版本号约定
- 遵循简易语义化：`主版本.功能版本.修订号`
- 框架/导航等结构性里程碑提升功能版本号
- 小修订提升修订号

## 4. 代码规范

- 技术栈固定：Python + PySide6 + SQLite
- 项目结构清晰分层，模块职责单一
- 代码注释完整（模块 docstring + 关键函数/类 docstring + 必要行内注释）
- 便于以后维护，遵循现有代码风格

## 5. 验证要求

- 每个任务完成后做冒烟测试，确保程序可启动、核心链路通。
- 测试通过后在回复中如实报告验证结果。

## 6. 自律原则

- 不臆测未给定的需求；有歧义时先与用户确认。
- 不为"以后可能用到"而提前实现功能。
- 完成一个任务就停，不连做多个任务。
